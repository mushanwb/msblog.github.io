lnmp搭建以及部署laravel

Posted on 2019-10-14 | In php

lnmp环境搭建

个人新手搭建lnmp环境，不用多说，一键安装包，傻瓜式安装。

lnmp一键安装包教程特别清晰，操作讲解到位，同时还介绍了lnmp和lnmpa的区别（这两个安装方式差不读，主要看区别，选择自己适合的）。

lnmp都清楚一点，主要说一下lnmpa的优势，复制一下介绍（lnmp教程里有说）：

LNMPA使用Nginx作为前端服务能够更快更及时的静态页面、js、图片等，当客户端请求访问动态页面时由Nginx的反向代理给Apache处理，Apache处理完再交予Nginx返回给客户端。
采用LNMPA能够更好的解决LNMP架构中由于PHP-FPM方面产生的502错误，同时能够为提供更安全的多用户多站点环境。

安装laravel

安装laravel最好看laravel官方文档laravel中文文档，很详细，简单说一下过程，以5.8为例。

cd 到项目目录（一般放/home/wwwroot/，随自己)

用composer创建项目，没有composer可以去官网下载安装composer安装，好像默认有安装composer：（可以更改项目名，只需要把blog换成你想要的项目名就可以）

composer create-project --prefer-dist laravel/laravel blog "5.8.*"

下载后进入项目 (默认项目名为blog，或者你在composer的时候自定义的项目名)

cd blog

给一些文件夹可写的权限，例如：storage 一般日志文件 和 bootstrap/cache 缓存文件,不然laravel无法运行

chmod -R 777 storage chmod -R 777 bootstrap

如果没有 .env 文件，但是有 .env.example 文件，将 .env.example 文件复制或重命名为 .env，如果有就不需要（一般用git是没有.env，被gitignore去掉了，后面会有git的安装方式）

cp .env.example .env

设置laravel密钥，主要 实现框架中的encrypt(加密)服务，例如存储用户的session，cookie等，从而确保信息安全。

php artisan key:generate

配置nginx（添加虚拟机）

进入一键包nginx的默认配置目录：

cd /usr/local/nginx/conf/

nginx的配置文件 nginx.conf ，一般我喜欢用虚拟机，配置虚拟机，进入虚拟机文件夹：

cd /usr/local/nginx/conf/ vhost

添加虚拟机配置，（注意：最好用命令去添加，不要复制粘贴别的配置，然后去改，原因后面会说到）

lnmp vhost add

laravel配置

到此，laravel部署完了，可以开是写项目了，我们可以简单配置一下laravel

1、更改时区：在config/app.php文件中，将timezone参数设置为PRC，北京时间

2、debug模式：在.env文件中，将APP_DEBUG参数改为true，默认是开启的，但是千万不要在正式项目中开启debug模式，否则报错的时候会暴露.env文件的配置

3、日志设置：在.env文件中，将LOG_CHANNEL参数改为daily，默认为stack，所有的日志都放在一个文件中，改为daily后，可以将日志分成每天的，会自动生成日志文件，文件名为当天时间，做多保留14天的日志，可以在config/logging.php里设置

4、数据库配置，在.env文件中，DB开头的参数，有默认给的值，只需要更改db_database数据库名称，db_username数据库账号，db_password数据库密码
