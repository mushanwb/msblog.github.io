lnmpa-用git部署laravel

Posted on 2019-10-15 | In linux , php

前言

前面有一篇文章已经大致介绍了lnmp和lnmpa的区别，同时也介绍了lnmp下载和laravel下载

参考lnmp搭建以及部署laravel，这里主要介绍一下lnmpa和lnmp的部署区别，同时介绍一下git部署代码。

lnmpa的区别

当lnmpa下载安装之后，先添加虚拟机，别着急去配置nginx，在lnmpa下，nginx只处理静态页面，而动态页面则是由apache处理，这时候并不用像lnmp那样在去添加nginx的其他配置。

当你用命令添加了虚拟机之后，不仅会在/usr/local/nginx/conf/vhost/目录下添加一个nginx配置文件，在/usr/local/apache/conf/vhost/目录下也会添加一个apache配置文件。所以添加完虚拟机之后就可以直接访问了，如果出现laravel首页则表示成功了。

出现500，网页无法正常运作

有时候可能添加完虚拟机后，访问出现 该网页无法正常运作 http error 500，这时候去php的配置文件，默认在/usr/local/php/etc/下的php.ini文件，进去搜索display_errors参数，将默认的Off改为On，就可以查看无法访问原因。记得改完重启lnmp。

一般出现的错误为：

**Warning**: require(): open_basedir restriction in effect. File(/项目路径/vendor/autoload.php) is not within the allowed path(s)

这是因为目录被限制访问了。

解决办法：上面说到，在你添加虚拟机的时候，默认会在/usr/local/apache/conf/vhost/目录下也会添加一个apache配置文件，vim 修改这个文件

将open_basedir 后面的文件目录下设置的public去掉，保存退出，重启lnmp即可

如果这时候出现500，但是没有出现错误，应该是laravel没有配置，下面说一下git部署laravel

git部署laravel

本地操作

在github上新建一个仓库，复制仓库地址，在本地存放项目目录里运行

git clone 复制的仓库地址

eg：git clone git@github.com:mushanwb/mswx.git，用ssh必须要把本地的私钥放到github配置中

进入本地克隆下来的项目目录，去laravel官网下载laravel

composer create-project --prefer-dist laravel/laravel blog "5.8.*"

下载完后运行：

git add .

git commit -m "add laravel"

git push -u origin master

就可以看到github仓库里有laravel文件了，在本地新建一个develop分支，并且将develop分支代码推到远程仓库

git checkout -b develop

git push -u origin develop

这样本地就会新建一个develop分支，并且将代码推送到远程的时候，远程仓库也会新建一个develop分支

服务器操作

进入服务器项目目录，克隆远程仓库代码

git clone 复制的仓库地址

这时候是没有拓展文件vendor以及一些配置文件.env的，使用composer update命令加载拓展文件

composer update 和 composer install 的区别

install：

install 命令从当前目录读取 composer.json 文件，处理了依赖关系，并把其安装到 vendor 目录下。

如果当前目录下存在 composer.lock 文件，它会从此文件读取依赖版本，而不是根据 composer.json 文件去获取依赖。这确保了该库的每个使用者都能得到相同的依赖版本。

如果没有 composer.lock 文件，composer 将在处理完依赖关系后创建它。

update ：

为了获取依赖的最新版本，并且升级 composer.lock 文件，你应该使用 update 命令。

这将解决项目的所有依赖，并将确切的版本号写入 composer.lock。

composer安装中出错

问题一：

proc_open()： 执行一个命令并打开文件指针用于读取以及写入。

该函数被php禁用了，找到php的配置文件php.ini，一般在/usr/local/php/etc/文件下，找到 disable_functions参数，删除里面的proc_open即可

问题二

proc_get_status()： 获取使用 proc_open() 所打开进程的信息。

同上面一样，函数被禁用，在php.ini中删除proc_get_status即可

问题三

切换国内镜像：

composer config -g repo.packagist composer https://packagist.phpcomposer.com

laravel配置

laravel配置在lnmp搭建以及部署laravel里面有说，这里简单提一下

一：将.env.example文件更改为.env文件

cp .env.example .env

二：配置laravel的key

php artisan key:generate

其实这两个配置配好了，laravel也能正常启动了，然后其他一些小配置在lnmp搭建以及部署laravel里面也有说。

附带：github配置私钥

查看本地私钥（服务器或者本机电脑）

cd ~/.ssh

cat id_rsa.pub

复制私钥，添加到github的私钥里面：个人中心 -> settings -> SSH and GPG keys -> new SSH key
